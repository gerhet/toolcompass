Understanding Base64 Encoding

Base64 is a binary-to-text encoding scheme that represents binary data as a printable ASCII string. Every 3 bytes of binary data become exactly 4 Base64 characters, producing output that is ~33 % larger than the original but safe to transmit over any text-based protocol — HTTP, JSON, XML, or email.

This free Base64 encoder and decoder runs entirely in your browser. No data is ever sent to a server, no account is required, and nothing is stored anywhere. Your files and text stay completely private on your device.

Example Conversion

Text: Hello World Base64: SGVsbG8gV29ybGQ= Text: ToolCompass Base64: VG9vbENvbXBhc3M=

Common Use Cases

  • Data URIs: Embed images directly in HTML or CSS without a separate HTTP request.
  • Email Attachments: MIME encoding transmits binary files safely over SMTP.
  • API Authentication: HTTP Basic Auth sends credentials as Base64-encoded username:password.
  • URL Parameters: URL-safe Base64 encodes binary tokens (JWTs, session IDs) without percent-encoding.
  • Clipboard / Storage: Serialise binary data to a string for localStorage or clipboard APIs.

Standard Base64 uses + and / which have special meaning in URLs. URL-safe Base64 replaces + with - and / with _ so the string can be used directly in query parameters or path segments without percent-encoding. Padding characters (=) are also removed.
